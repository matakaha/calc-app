# 非機能要件定義書

## 1. 技術スタック

| 項目 | 技術 |
|---|---|
| 開発言語 | JavaScript（TypeScript は使用しない） |
| フレームワーク | React（Node.js） |
| テスト | Playwright（E2Eテストのみ。ユニットテストは対象外） |

## 2. ホスティング・インフラ

| 項目 | 内容 |
|---|---|
| ホスティングサービス | Azure Static Web Apps |
| プラン | Standard（標準） |
| リージョン | East Asia |
| カスタムドメイン | 不使用（Azure 提供のデフォルトドメインを使用） |

## 3. パフォーマンス

- 本アプリケーションは **デモサイト** としての位置づけである。
- そのため、ページ読み込み時間・レスポンス時間等の **パフォーマンス目標値は設定しない**。

## 4. ブラウザ対応

| 対象ブラウザ | バージョン |
|---|---|
| Microsoft Edge | 最新版のみ |

- 上記以外のブラウザ（Chrome, Firefox, Safari 等）での動作は保証しない。

## 5. セキュリティ

- **認証・認可は不要** とする。
- 誰でもアクセス可能な公開サイトとして運用する。

## 6. アクセシビリティ

- WCAG 等のアクセシビリティ基準への準拠は、**現時点では不要** とする。
- 将来的な対応は別途検討する。

## 7. CI/CD パイプライン

- **GitHub Actions** を使用した CI/CD パイプラインを構築する。
- プルリクエスト作成時に Playwright による E2E テストを自動実行する。
- 対象ブランチへのマージ・プッシュ時に自動デプロイを実行する。

## 8. ブランチ戦略・環境

### 8.1 ブランチ運用

| ブランチ | 用途 |
|---|---|
| `main` | 開発のベースブランチ |
| `topic/*` | `main` から作成する作業ブランチ。機能開発・修正を行う |
| `release/stage` | ステージング環境用ブランチ。一定の作業完了後に作成する |
| `release/prod` | 本番環境用ブランチ。本番リリース時に作成する |

### 8.2 デプロイ環境

| 環境 | デプロイ契機 | 説明 |
|---|---|---|
| ステージング（プレビュー） | `release/stage` ブランチの作成・更新 | Azure Static Web Apps のプレビュー環境を利用 |
| 本番 | `release/prod` ブランチの作成・更新 | Azure Static Web Apps の本番環境へデプロイ |
