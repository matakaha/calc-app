# 機能要件定義書

## 1. 概要

四則演算（加算・減算・乗算・除算）が可能な Web 電卓アプリケーションを作成する。

## 2. 画面仕様

### 2.1 ディスプレイ

- 計算結果および入力値を表示するディスプレイを画面上部に配置する。
- ディスプレイの表示文字数は **最大9文字** とする。
  - 正の整数: 最大9桁（例: `123456789`）
  - 負の整数: 最大8桁（マイナス記号で1文字使用。例: `-12345678`）
  - 小数を含む場合: 小数点で1文字使用するため、表示可能な数字の桁数がさらに1桁減る（例: `1234567.8`, `-123456.7`）
- 初期状態では `0` を表示する。

### 2.2 レスポンシブデザイン

- モバイル端末での利用を想定し、レスポンシブデザインに対応する。
- 画面幅に応じてボタンサイズ・レイアウトが適切に調整されること。

### 2.3 入力方式

- マウスクリックおよびタッチ操作による入力のみをサポートする。
- キーボード入力には対応しない。

## 3. ボタン仕様

電卓には以下のボタンを配置する。

| ボタン | 説明 |
|---|---|
| `0` 〜 `9` | 数字入力 |
| `.` | 小数点入力 |
| `+` | 加算 |
| `-` | 減算 |
| `×` | 乗算 |
| `÷` | 除算 |
| `=` | 計算実行 |
| `AC` | All Clear（全入力・演算状態をリセットし `0` に戻す） |
| `DEL` | Delete（最後の1文字を削除するバックスペース動作） |
| `+/−` | 表示中の数値の符号を反転（正負切替） |

## 4. 計算仕様

### 4.1 基本動作

- 四則演算（`+`, `-`, `×`, `÷`）を実行できること。
- 小数点の入力・計算に対応すること。

### 4.2 表示桁数と切り捨て

- 計算結果が9文字に収まらない場合は、収まらない部分を **切り捨て** て表示する。

### 4.3 演算子の連続入力

- 演算子が連続して入力された場合は、**最後に入力された演算子のみを有効** とする。
  - 例: `5` → `+` → `-` と入力した場合、有効な演算子は `-` となる。

### 4.4 エラー処理

| エラー種別 | ディスプレイ表示 |
|---|---|
| ゼロ除算（0で割る操作） | `ZERO DIV` |
| その他のエラー | `ERROR` |

- エラー表示中は、`AC` キーの押下で初期状態（`0` 表示）に復帰する。

## 5. スコープ外（非対応機能）

以下の機能は本アプリケーションのスコープ外とし、実装しない。

- 計算履歴の保持・表示
- キーボード入力のサポート
- 括弧演算（`(`, `)`）
- 関数演算（sin, cos, tan, log 等）
- パーセント演算（`%`）
- メモリ機能（M+, M-, MR, MC）
